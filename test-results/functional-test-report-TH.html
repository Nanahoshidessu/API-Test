4 รายการปัญหาที่พบ (Defect Register)
DEF-001
จำนวน ArrCategories ไม่ตรงกับ specification สำหรับ Type=Day
MEDIUM
รายการรายละเอียด
Endpoint ที่ได้รับผลกระทบ/GetDataMeterSum, /GetDataMultimeter, /GetDataMeterDetail
Test CasesMS-F-01, MM-F-01, MD-F-01, RF-01, RF-02, RF-05, RF-08a, RF-08b (รวม 8 กรณี)
คำอธิบายเมื่อใช้ Type=Day API คืนจำนวน time-slot ใน ArrCategories น้อยกว่าที่ spec กำหนด สำหรับ Interval=15 นาที ต้องการ 96 รายการ (24 ชั่วโมง × 4) แต่ได้รับ 93 รายการ สำหรับ Interval=5 นาที ต้องการ 288 รายการ (24 ชั่วโมง × 12) แต่ได้รับ 277 รายการ
ExpectedInterval=15 → 96 entries | Interval=5 → 288 entries
ActualInterval=15 → 93 entries | Interval=5 → 277 entries
สมมติฐานสาเหตุAPI อาจสร้าง categories เฉพาะ time slot ที่มีข้อมูลจริง แทนที่จะสร้าง grid ตลอด 24 ชั่วโมง
DEF-002
Dictionary keys ใช้ numeric index แทน DataSelect values
HIGH · CRITICAL PRIORITY
รายการรายละเอียด
Endpoint ที่ได้รับผลกระทบ/GetDataMeterSum
Test CasesMS-F-02, MS-F-03, MS-F-04, MS-F-05, RF-03 (รวม 5 กรณี)
คำอธิบายสำหรับ Type=Week, Month, Year, Max10Year ค่า key ใน ArrDataTarget และ ArrDataActual เป็น numeric index ('0', '1', '2') แทนที่จะเป็นค่า DataSelect จริง ทำให้ client ไม่สามารถ map ข้อมูลกราฟกับ label ที่ถูกต้องได้
ตัวอย่าง — MonthExpected keys: '04', '05', '06' → Actual: '0', '1', '2'
ตัวอย่าง — WeekExpected: '2025-06-02||2025-06-08' → Actual: '0', '1'
สมมติฐานสาเหตุAPI serialize response เป็น ordered array และใช้ sequential integer keys แทนที่จะ preserve DataSelect strings เป็น key
DEF-003
MeterLabelMap — ไม่พบ field ชื่อ meter ID ใน response
MEDIUM
รายการรายละเอียด
Endpoint ที่ได้รับผลกระทบ/GetDataMeterDetail
Test CasesMD-F-01, MD-F-02, MD-F-03, MD-F-04, RF-05, RF-06, RF-07 (รวม 7 กรณี)
คำอธิบายScript ตรวจหา meter identifier ใน MeterLabelMap จาก field ชื่อ id, Id, Meter, meter แต่ไม่พบ field ใดเลย ต้องยืนยันชื่อ field จริงจาก API response
Expectedแต่ละรายการใน MeterLabelMap มี field id หรือ Id ที่ตรงกับ meter ที่ request
Actualไม่พบ field ดังกล่าว — ได้ undefined ทั้งหมด
การแก้ไขตรวจสอบ response จริงเพื่อหาชื่อ field ที่ถูกต้อง แล้วอัปเดต assertion ใน script
DEF-004
HTTP 500 Internal Server Error — /GetDataMeterDetail Type=Max10Year
HIGH · CRITICAL PRIORITY
รายการรายละเอียด
Endpoint ที่ได้รับผลกระทบ/GetDataMeterDetail
Test CasesMD-F-05
คำอธิบายเมื่อส่ง type=Max10Year ไปยัง /GetDataMeterDetail server คืน HTTP 500 Internal Server Error — เป็นสภาวะ crash ของ server สำหรับ input ชุดนี้
Request ที่ทำให้เกิดปัญหาPOST /GetDataMeterDetail body: {"type":"Max10Year","Meter":[1],"DataSelect":"2025|2016"}
ExpectedHTTP 200 พร้อม chart series response
ActualHTTP 500 Internal Server Error
สมมติฐานสาเหตุType Max10Year อาจยังไม่ได้ implement หรือมี unhandled exception ใน MeterDetail handler
