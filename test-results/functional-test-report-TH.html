4 รายการปัญหาที่พบ (Defect Register)
ปัญหาทั้งหมดที่พบใน local server (v1.0–v1.2) ไม่ปรากฏ บน remote deployment นี้ — DEF-001, DEF-002, DEF-003 ถูกปิดแล้ว
DEF-001
จำนวน ArrCategories ไม่ตรงกับ specification สำหรับ Type=Day
CLOSED · RESOLVED
รายการรายละเอียด
สถานะClosed — ไม่พบปัญหานี้บน remote server (v1.3, 5 พฤษภาคม 2569) — ArrCategories คืนค่าครบถ้วนตาม spec แล้ว
Endpoint ที่ได้รับผลกระทบ (เดิม)/GetDataMeterSum, /GetDataMultimeter, /GetDataMeterDetail
Test Cases ที่ผ่านแล้วMS-F-01, MM-F-01, MD-F-01, RF-01, RF-02, RF-05, RF-08a, RF-08b (รวม 8 กรณี)
คำอธิบายปัญหาเดิมLocal server คืน 93 entries สำหรับ Interval=15 (ต้องการ 96) และ 277 สำหรับ Interval=5 (ต้องการ 288) — Remote server คืนค่าครบถ้วนแล้ว

Dictionary keys ใช้ numeric index แทน DataSelect values
CLOSED · RESOLVED

สถานะClosed — ไม่พบปัญหานี้บน remote server (v1.3, 5 พฤษภาคม 2569) — Dictionary keys ใช้ DataSelect strings ถูกต้องแล้ว
Endpoint ที่ได้รับผลกระทบ (เดิม)/GetDataMeterSum
Test Cases ที่ผ่านแล้วMS-F-02, MS-F-03, MS-F-04, MS-F-05, RF-03 (รวม 5 กรณี)
คำอธิบายปัญหาเดิมLocal server ใช้ '0', '1', '2' เป็น key แทน '04', '05', '06' — Remote server คืน DataSelect strings ถูกต้องแล้ว
DEF-003
MeterLabelMap — format ไม่ตรงกับ specification
CLOSED · RESOLVED
รายการรายละเอียด
สถานะClosed — ไม่พบปัญหานี้บน remote server (v1.3, 5 พฤษภาคม 2569) — MeterLabelMap คืน format ที่ assertion ตรวจได้สำเร็จ

Test Cases ที่ผ่านแล้วMD-F-01, MD-F-02, MD-F-03, MD-F-04, MD-F-05, RF-05, RF-06, RF-07 (รวม 8 กรณี)
คำอธิบายปัญหาเดิมLocal server คืน [{"1": "1"}] แทน [{"id": "1", "label": "..."}] — Remote server คืน format ที่ assertion ตรวจพบ meter ID ได้สำเร็จ
DEF-004
HTTP 500 — /GetDataMeterDetail Type=Max10Year (Workaround ยังคงอยู่)
MITIGATED · VERIFY REQUIRED
รายการรายละเอียด
สถานะWorkaround ยังคงอยู่ — Test script ส่ง single-year DataSelect และได้ HTTP 200 บน remote server — ยังไม่ได้ยืนยันว่า server-side fix ถูกแก้ไขแล้วหรือไม่
Endpoint ที่ได้รับผลกระทบ/GetDataMeterDetail
Root Cause (ยืนยันแล้ว)DataMeterDetailController.cs บรรทัด 288: Int32.Parse(DataSelect) ทำงานก่อนเช็ค Type — เมื่อ DataSelect มีอักขระ | จะ throw System.FormatException
Workaround ที่ใช้Test script MD-F-05 ส่ง "{{scenario_year}}" แทน "{{scenario_year}}|{{scenario_year_start}}" → MD-F-05 ผ่านแล้ว
การดำเนินการที่แนะนำทดสอบด้วย DataSelect รูปแบบเดิม "year|year_start" บน remote server เพื่อยืนยันว่า server-side fix ถูกแก้ไขแล้ว
